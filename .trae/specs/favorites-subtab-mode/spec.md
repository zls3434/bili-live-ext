# 收藏夹子标签模式 Spec

## Why
当前收藏界面以卡片列表方式展示所有收藏夹，点击后进入收藏夹内部视频列表，但没有返回收藏夹列表的交互路径，且每次进入收藏界面都从收藏夹列表开始，无法记忆上次打开的收藏夹。需要改为子标签模式，让用户在各收藏夹之间快速切换，同时支持记忆上次打开的收藏夹和懒加载视频列表。

## What Changes
- **收藏界面从卡片列表模式改为子标签模式**：收藏夹不再以卡片形式展示，而是显示为水平子标签栏 + 下方视频列表
- **初次进入默认打开默认收藏夹**：首次进入收藏界面时，自动选中第一个收藏夹（默认收藏夹）并加载其视频列表
- **切换子标签切换视频列表**：点击子标签切换收藏夹时，清空当前视频列表并加载目标收藏夹的视频
- **记忆上次打开的收藏夹**：使用 VSCode globalState 持久化存储上次打开的收藏夹 ID，下次进入收藏界面时自动打开该收藏夹
- **收藏夹视频列表懒加载**：滚动到底部时自动加载更多视频，复用现有的滚动加载机制

## Impact
- Affected specs: 收藏界面视图逻辑
- Affected code:
  - `src/webview/htmlTemplate.ts` — 前端渲染逻辑、CSS 样式、子标签切换事件处理
  - `src/webview/BiliMainViewProvider.ts` — 收藏夹相关消息处理、持久化存储上次收藏夹 ID
  - `src/webview/viewDataLoader.ts` — 收藏夹视频列表数据加载逻辑改造（不再只加载列表，还需加载默认收藏夹视频）
  - `src/types/index.ts` — 可能需要新增 ContentView 枚举值或调整现有逻辑

## ADDED Requirements

### Requirement: 收藏夹子标签模式
系统 SHALL 在收藏界面以子标签栏方式展示所有收藏夹，替代现有的卡片列表模式。

#### Scenario: 初次进入收藏界面
- **WHEN** 用户首次进入收藏界面（无历史记录）
- **THEN** 显示收藏夹子标签栏，默认选中第一个收藏夹子标签，并自动加载该收藏夹的视频列表

#### Scenario: 再次进入收藏界面
- **WHEN** 用户再次进入收藏界面（有历史记录）
- **THEN** 显示收藏夹子标签栏，自动选中上次打开的收藏夹子标签，并加载该收藏夹的视频列表

#### Scenario: 切换收藏夹子标签
- **WHEN** 用户点击另一个收藏夹子标签
- **THEN** 清空当前视频列表，选中目标子标签，显示加载状态，加载目标收藏夹的视频列表
- **AND** 将目标收藏夹 ID 保存到 VSCode globalState 中作为"上次打开的收藏夹"

#### Scenario: 收藏夹视频列表懒加载
- **WHEN** 用户滚动到视频列表底部附近（距离底部 50px 以内）
- **THEN** 自动加载下一页视频并追加到视频列表末尾
- **AND** 直到 `hasMore` 为 false 时停止加载

#### Scenario: 收藏夹只有一个收藏夹
- **WHEN** 用户的收藏夹列表只有一个收藏夹
- **THEN** 仍然显示子标签栏（只有一项），下方显示该收藏夹的视频列表

#### Scenario: 收藏夹为空
- **WHEN** 用户没有任何收藏夹
- **THEN** 显示空状态提示"暂无收藏夹"

### Requirement: 收藏夹 ID 持久化
系统 SHALL 将用户上次打开的收藏夹 ID 持久化存储到 VSCode globalState 中。

#### Scenario: 用户切换收藏夹
- **WHEN** 用户点击不同的收藏夹子标签
- **THEN** 将选中的收藏夹 ID 写入 globalState（key: `lastFavoriteFolderId`）

#### Scenario: 用户进入收藏界面
- **WHEN** 用户切换到收藏 Tab
- **THEN** 读取 globalState 中的 `lastFavoriteFolderId`，如果存在且在当前收藏夹列表中，则自动选中该子标签；否则选中第一个收藏夹

### Requirement: 收藏夹子标签样式
系统 SHALL 使用与关注子Tab一致的视觉风格渲染收藏夹子标签栏。

#### Scenario: 子标签栏样式一致性
- **WHEN** 渲染收藏夹子标签栏
- **THEN** 复用 `.follow-sub-tab` 的样式规则，保持视觉一致性
